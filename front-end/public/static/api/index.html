API library documentation

List of exiting APIs for CookedSpecially

  • getMenus Used when the application starts
  • getTables Used to display the list of tables
  • setTable Used by the Save button in the admin panel
  • getCheck Used to get a table's check data
  • addToOrder In progress...

getMenus.jsp

Description:
This API is used when the application starts-up and returns the data to populate all menus and menus items for the restaurant.
(Right now I am using a proxy server to get the menu json feed from the original getRestaurantMenus.jsp API)

Arguments:
- restaurantId: Restaurant ID from MenuMaker app.

Usage:
/api/getMenus.jsp?restaurantId=<id>

Run Code

getTables.jsp

Description:
This API is used to display the list of tables list for a restaurant in the Control Panel window-- it returns all tables for a restaurant.
(In the future, it *could* be called with different arguments (restaurantId, userName, userId, ...)

Arguments:
- restaurantId: restaurantId.

Usage:
/api/getTables.jsp?restaurantId=<id>

Run Code

setTable.jsp

This API is used to set the table status in the database to:
- Busy
- Available
- Unavailable

Arguments:
- tableId: tableID.
- status: Busy | Available | Unavailable

Usage:
/api/setTable.jsp?restaurantId=<id>&tableId=<id>&status=<status>

Run Code

getCheck.jsp

This API is used when a new or existing table is set in the device, or when the user clicks on the "View My Check" button.
- If the table has an open check,the API returns the order data.
- if the table has no open check, the API returns a new (blank) check id

Arguments:
- restaurantId: restaurantId.
- tableId: tableID.

Usage:
/api/getCheck.jsp?restaurantId=<id>&tableId=<id>
